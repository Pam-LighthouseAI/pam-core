Instance Files (Your Bot)

📄 config.json

C:\nanobot\instance3\config.json

The brain's configuration. Defines:

  • model - Which LLM to use
  • api_key - Authentication
  • port - Gateway port
  • channels - Telegram, WhatsApp, etc.
  • providers - API endpoints

🧠 SOUL.md

C:\nanobot\instance3\workspace\SOUL.md

The personality and behavior instructions. Loaded into every LLM call.

  • Who I Am - Identity
  • How I Talk - Communication style
  • How I Work - Action directives
  • Never / Always - Ironclad rules

📚 MEMORY.md

C:\nanobot\instance3\workspace\memory\MEMORY.md

Long-term facts loaded into every conversation.

  • User preferences
  • Project context
  • Important relationships
  • Technical decisions

📜 HISTORY.md

C:\nanobot\instance3\workspace\memory\HISTORY.md

Append-only log of past events. Not loaded into context - grep searchable.

  • Conversation summaries
  • Task completions
  • Important decisions

🔧 Skills

C:\nanobot\instance3\workspace\skills\{name}\SKILL.md

Modular capabilities the agent can learn.

  • memory - Memory management
  • cron - Scheduling
  • weather - Weather data
  • goal-tracker - Project tracking

💬 Sessions

.nanobot/sessions/{session_key}.json

Conversation history per channel/chat.

  • Message history
  • Tools used
  • Timestamps
  • Consolidation state

Key Classes

AgentLoop

agent/loop.py

The brain's processing engine. Orchestrates everything:

  • Receives messages from bus
  • Builds context via ContextBuilder
  • Calls LLM via LiteLLMProvider
  • Executes tools via ToolRegistry
  • Sends responses back to bus

ContextBuilder

agent/context.py

Assembles the system prompt:

  • Loads SOUL.md, USER.md, AGENTS.md
  • Loads MEMORY.md
  • Loads always-skills
  • Builds skills summary
  • Adds session history

LiteLLMProvider

providers/litellm_provider.py

Talks to the LLM:

  • Resolves model name
  • Applies provider prefixes
  • Handles auto-routing to free models
  • Parses tool calls
  • Returns LLMResponse

TelegramChannel

channels/telegram.py

Telegram interface:

  • Polls for messages
  • Downloads media
  • Transcribes voice
  • Sends typing indicator
  • Converts markdown to HTML

Step-by-Step: What Happens When You Send a Message

Startup: nanobot gateway --port 18792

The CLI command in cli/commands.py creates:
• MessageBus - message queues
• LiteLLMProvider - LLM connection
• AgentLoop - processing engine
• ChannelManager - starts Telegram, etc.
• CronService - scheduled tasks

Message Arrives: TelegramChannel._on_message()

When you send a message in Telegram:
• telegram.py receives it via polling
• Downloads any media (photos, voice)
• Transcribes voice messages
• Creates InboundMessage
• Publishes to MessageBus

Agent Picks Up: AgentLoop.run()

The agent loop is waiting on the bus:
• msg = await self.bus.consume_inbound()
• Calls _process_message(msg)

Build Context: ContextBuilder.build_messages()

Assembles the prompt for the LLM:
• build_system_prompt() loads SOUL.md, MEMORY.md, skills
• Adds session history from SessionManager
• Adds current user message
• Returns [system_msg, history..., user_msg]

Call LLM: LiteLLMProvider.chat()

Sends to OpenRouter/Anthropic/etc:
• Resolves model name (adds provider prefix)
• Checks auto-router for free model routing
• Calls litellm.acompletion()
• Parses response into LLMResponse

Tool Execution: ToolRegistry.execute()

If LLM returns tool calls:
• Agent loops through response.tool_calls
• Each tool executes (read_file, exec, etc.)
• Results added back to messages
• Calls LLM again with tool results
• Repeats until no more tool calls

Send Response: MessageBus.publish_outbound()

Final response goes back:
• Agent creates OutboundMessage
• Publishes to bus
• TelegramChannel.send() picks it up
• Converts markdown to HTML
• Sends via Telegram API

Memory Consolidation: _consolidate_memory()

When session gets long:
• Summarizes old messages via LLM
• Appends summary to HISTORY.md
• Extracts facts to MEMORY.md
• Clears old messages from session

What Talks to the LLM

Direct LLM Calls

These components call the LLM directly:

  • AgentLoop - Main chat responses
  • _consolidate_memory() - Memory summarization
  • SubagentManager - Background tasks

What's in Every LLM Call

The system prompt always includes:

  • Identity - "You are nanobot..."
  • SOUL.md - Personality
  • MEMORY.md - Long-term facts
  • Always Skills - Pre-loaded capabilities
  • Skills Summary - Available skills list
  • Session History - Recent messages

Tool Definitions

Tools are sent with each LLM call:

  • read_file - Read file contents
  • write_file - Write to file
  • edit_file - Edit file
  • list_dir - List directory
  • exec - Run shell command
  • web_search - Search the web
  • web_fetch - Fetch URL
  • message - Send message
  • spawn - Create subagent
  • cron - Schedule task
